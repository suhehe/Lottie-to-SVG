# Lottie Compiler

一个本地可用的 `Lottie JSON -> 原生 SVG 动画` 转换工具。

它提供一套可视化工作台，用来上传 Lottie JSON、预览原始动画、编译 SVG、查看源码、切换压缩输出并导出结果。相比“只导出静态 SVG”或“只截图”的方案，这个工具更关注动画结构还原、源码可读性和可交付性。

## 功能概览

- 上传本地 `.json` Lottie 文件并自动解析
- 预览原始 Lottie 动画
- 查看原始 JSON 源码
- 编译并预览原生 SVG 动画
- 查看 SVG 源码
- 显示 JSON 与 SVG 文件大小
- 默认启用 SVG 压缩开关
- 导出当前版本的 SVG
- 自动统计素材元信息
- 自动分析当前素材的支持特性与未支持特性

## Cloudflare 部署

当前项目已经是 Vite 应用，Cloudflare 必须发布构建产物 `dist`，不能直接托管仓库根目录。

错误部署方式的典型现象是：

- 页面样式正常
- 但脚本没有执行
- 上传区域点击无反应
- 页面一直停留在未初始化的大上传区

根因是源码入口 [`index.html`](./index.html) 直接引用了 ESM 脚本，而源码中的依赖包含裸模块导入，例如 `lottie-web`。这些导入必须先经过 Vite 打包，浏览器静态托管源码时不会解析。

Cloudflare 正确配置：

- Build command: `npm run build`
- Build output directory: `dist`

如果使用 `wrangler`，仓库内的 [`wrangler.jsonc`](./wrangler.jsonc) 已经配置为发布 `dist`。

## 已修复的典型问题

这版工具已经针对一些真实素材中的高频问题做过修复，例如：

- 复合路径导致的镂空丢失
- 多 fill / multi-pass 绘制遗漏
- 非法 `keySplines` 导致的图层消失
- 静态 transform 合并顺序错误导致的图层偏移
- 压缩后透明度窗口动画异常
- 3D layer 压扁近似导致的影子变形

## 当前限制

这个工具不是完整的 Lottie 渲染器，仍然有一部分特性不会 100% 还原。当前主要限制包括：

- `Trim Paths`
- `Merge Paths`
- 反向 `Track Matte`
- `Masks`
- `Time Remap`
- 混合模式
- 径向渐变目前不会按原生径向方式完整输出
- 某些复杂的 AE / Lottie easing 会自动回退为更安全的采样动画

这套策略是有意为之：优先保证结果稳定、可预览、可导出、尽量不丢内容，而不是为了压缩体积去走高风险转换。

## 压缩策略

SVG 面板中的“压缩”开关默认开启。

压缩版不是重新渲染一套动画，而是在原始 SVG 基础上做安全压缩，常见处理包括：

- 压缩标签间空白
- 缩短部分颜色表达
- 最小化路径数字
- 去掉部分默认值属性
- 在不改变动画语义的前提下收缩输出字符串

如果压缩版并没有比原始版更小，工具会自动回退显示原始版，避免“越压越大”。

## 开发说明

如果你准备继续开发，建议优先关注这些方向：

- 提升更多复杂 Lottie 特性的覆盖率
- 继续优化 transform / path 动画体积
- 增强浏览器级回归验证

## 注意事项

- 原始 Lottie 预览依赖 `lottie-web`
- 某些复杂动画会为了稳定性自动回退到采样输出，因此文件体积可能仍然偏大
- 这个工具更适合“设计还原、交付导出、代码检查、结构分析”，不适合作为完整的 Lottie 运行时替代品
